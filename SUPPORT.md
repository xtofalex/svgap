# Support

SV-Gap is early research software maintained on a best-effort basis.

- Use GitHub Discussions for integration questions and taskpack ideas.
- Use the [collaboration intake](https://github.com/shsridhar-beep/svgap/issues/new?template=collaboration.yml)
  when you have a research-to-production handoff question but do not yet know
  whether it should become a model run, task, integration, or contribution.
- Use the issue templates for reproducible bugs, backend proposals, structural
  false results, and taskpacks.
- Report active security vulnerabilities privately as described in
  `SECURITY.md`.

The project does not provide silicon-signoff advice or support proprietary EDA
tools. Production users remain responsible for their approved verification and
signoff flows.
