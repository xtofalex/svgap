# SV-Gap v0.3.0-alpha.1

This adoption-focused prerelease makes the research-to-production trust gap
visible in one command and opens concrete collaboration paths for frontier-model
researchers, chip-design AI builders, benchmark maintainers, and EDA tool
authors.

## Highlights

- `svgap demo` produces a controlled functional-pass/structural-fail witness in
  under two minutes and can preserve a complete machine-readable reproducer.
- A bring-your-own-RTL tutorial covers intent declaration, imported functional
  evidence, evaluation, explanation, and CI export.
- The first schema-validated public result registry spans generation,
  diagnosis, and repair profiles.
- An exploratory two-configuration frontier-model baseline exposes a crossed
  failure mode: preserving uncertainty and repairing the configured finding are
  separate capabilities.
- A Material for MkDocs site, visual explainer, contributor issues, and seeded
  research discussions provide clear entry points for users and collaborators.
- PyPI trusted publishing and the open-tool container reduce installation
  friction.

## Claim boundary

The baseline is one prompt and one run for two configurations from one provider.
It demonstrates the evaluation contract; it is not a model ranking or
population estimate. Structural results describe the configured open-source
backend and are not silicon signoff.

The real reset-release perturbation instrumenter remains unimplemented pending
patent and employer review.
