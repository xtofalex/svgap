# SV-Gap v0.3.0-alpha.5

This usability hotfix makes every public first-time path terminate in working
proof, actionable setup guidance, or a concrete collaboration route.

## First-time experience

- A zero-install controlled result and public model profile precede setup.
- The pinned container demo is the primary executable quickstart.
- The README uses canonical links that work from GitHub and PyPI.
- Current coverage, unsupported scope, platform support, local/no-telemetry
  behavior, untrusted-input guidance, and accountable maintenance are explicit.
- GitHub intake and maintainer email provide public and no-GitHub-account
  research-call paths. Do not send confidential artifacts through either.

## Actionable diagnostics

When Yosys or Icarus Verilog is missing, `svgap doctor` now prints:

- Homebrew remediation on macOS;
- `apt`, `dnf`, and `pacman` recipes on Linux;
- Docker Desktop or WSL2 guidance for Windows;
- the pinned no-host-install container command; and
- the troubleshooting documentation URL.

## Research conversion

The research-call intake, bounded design-partner workflow, and one-page
experiment contract define a legitimate path from practitioner interest to an
attributable protocol, evidence, validation, analysis, documentation, or code
contribution. A call or affiliation alone is not contributor status.

SV-Gap remains early digital-RTL research software. Its configured evidence
profiles are not certification, a general model ranking, or silicon signoff.

The version-specific archival DOI and immutable container digest will be added
after publication and independent verification.
